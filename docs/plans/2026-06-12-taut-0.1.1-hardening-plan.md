# Taut 0.1.1 Hardening Plan

Date: 2026-06-12
Status: Complete through release commit and local git tag. PyPI publication
is still externally blocked by the package-name request.

## 1. Goal

Close the gap between the shipped 0.1.0 and the project's own documents:
fix the one open implementation defect (handle quality), finish the
[TAUT-11] proof obligations, make the human rendering match the README
contract, run the overdue independent review, and tag 0.1.1. PyPI publish
remains blocked externally (name request pending) and is out of scope.

  (surface contract), [TAUT-11] (verification expectations)

Hardening runbook: **not triggered** — no new persistence, async,
contract surface, or one-way door in this slice. Rationale recorded here
per [DOM-5].

## 3. Invariants

All foundation-plan invariants (I1–I9) remain binding as amended by this
slice: I8 now allows exactly `simplebroker` plus `psutil`, with `psutil`
scoped to identity capture. Specifically in range here: I5 (write path),
I6 (cursor monotonicity — the new
concurrency tests protect it), I7 (the `--json` fields and exit codes do
not move while the *human* rendering changes — human output is explicitly
not a compatibility surface, [TAUT-8.2]).

## 4. Tasks

1. Handle quality fix (appendix item 3 — the prescription is there;
   this is a pointer, not a restatement).
   - Files: `taut/identity.py`, `tests/test_identity.py`.
   - Use `psutil` for untruncated process argv/executable/cwd capture
     where available. Keep the fallback parser correct: reconstruct
     `argv` from `ps` output without whitespace-splintering by
     incrementally joining tokens while the joined prefix is an existing
     path for `argv[0]`; Linux `/proc` is already NUL-correct.
   - Extend both existing identity tests to assert the resolved *handle*
     equals the durable ancestor's true basename (e.g. not
     `application`); keep the anchor-location assertions.
   - Done: the two-shell probe yields a real name; suite green.
2. [TAUT-11] burndown.
   - Files: `tests/test_watcher.py`, `tests/test_client.py` or new
     `tests/test_concurrency.py`.
   - Add: (a) live watcher vs ≥2 concurrent writer *processes* — no
     message lost, ordering by ts holds; (b) mid-watch **join** picked
     up via `add_queue` (mirror of the existing leave tests); (c) idle
     no-busy-spin bound (assert bounded poll/fetch counts while queues
     are drained, not wall-clock); (d) token acts-as from an unrelated
     process tree if not already covered in the 6a set.
   - Done: every [TAUT-11] bullet maps to a named test.
3. Renderer conformance (the one real decision: the README wins).
   - Files: `taut/cli.py`, `tests/test_cli.py`, README only if a detail
     is genuinely better changed there (then change the README first —
     it is the contract, [DOM-6] discipline applies in miniature).
   - Target format, exactly as the README shows: `read`/`log`/`watch`
     group by thread under a `── <thread> ──…` rule line; message rows
     `  HH:MM <from>  <text>` with notices rendered dimmed as
     `  HH:MM · <text>`; `-t` prepends the full id column; `list` shows
     `<thread>  <N> unread` with N from a bounded peek (cap display at
     `999+`), per the [TAUT-7.3] presentation note. Times are local
     HH:MM display only — storage stays broker timestamps ([TAUT-3.5]).
   - Done: the README Quick Start transcript reproduces byte-shape
     (modulo timestamps/ids) in a fresh directory.
4. Independent review, round 5 (overdue §8 checkpoints, combined).
   - Panel: codex + grok over the *implementation*: spec-conformance
     walk of [TAUT-8.1]/[TAUT-8.2] row by row against the real CLI,
     invariant greps, test-quality pass against [TAUT-11].
   - Findings recorded in the foundation plan's appendix as Round 5;
     authoring agent answers each point per [DOM-11].
5. Tag 0.1.1.
   - Task 9 release checklist executed (spec-conformance pass,
     `SchemaVersionError` gate proven, README claims audit); CHANGELOG
     entry; `uv build`; git tag. No PyPI step.
